## 🎯 核心功能模块

### 1. 插件注册与初始化
- **类名**: `DailyAINewsPlugin`
- **注册装饰器**: `@register()` 定义插件基本信息
- **初始化方法**: `__init__()` 设置数据目录、文件路径、订阅集合
- **启动方法**: `initialize()` 加载数据文件，启动定时任务

### 2. RSS订阅管理
- **RSS源**: 固定使用橘鸦AI日报RSS (`https://imjuya.github.io/juya-ai-daily/rss.xml`)
- **获取方法**: `_fetch_rss_latest()` 异步获取最新文章
- **XML解析**: 使用 `xml.etree.ElementTree` 解析RSS内容
- **日期解析**: `_parse_article_date()` 多策略解析文章日期

### 3. AI智能总结
- **总结Prompt**: 定义专业的AI资讯编辑角色和总结要求
- **AI调用**: `_summarize_with_ai()` 使用AstrBot内置LLM
- **内容处理**: 超长内容截断、HTML标签清理
- **回退机制**: AI失败时使用原文摘要

### 4. 定时推送系统
- **定时循环**: `_schedule_loop()` 每天设定时间检查RSS
- **补偿机制**: `_startup_compensation_check()` 启动时检查遗漏推送
- **轮询模式**: RSS未更新时定时重新检查
- **推送执行**: `_do_push()` 向所有订阅目标发送消息

### 5. 订阅管理
- **指令订阅**: `/ainews_sub` 和 `/ainews_unsub` 管理会话订阅
- **配置订阅**: 支持配置文件中的群聊和私聊列表
- **目标合并**: `_get_all_targets()` 合并指令和配置订阅
- **格式支持**: 跨平台格式 `{机器人名称}:{ID}`

### 6. 数据持久化
- **订阅数据**: `subscriptions.json` 存储指令订阅列表
- **推送记录**: `sent_news.json` 存储已推送的日期和文章链接
- **总结缓存**: `summary_cache.json` 缓存AI总结结果
- **原子写入**: `_atomic_write()` 确保数据完整性
- **文件锁**: `_file_lock` 防止并发写入冲突

### 7. 内容格式化
- **AI总结格式**: `_format_summary()` 专业的新闻推送格式
- **回退格式**: `_format_fallback()` 原文摘要格式
- **HTML清理**: `_clean_html()` 移除HTML标签和实体

## ⚙️ 配置系统

### 配置参数（_conf_schema.json）
- `push_hour`: 推送时间（小时，0-23）
- `push_minute`: 推送时间（分钟，0-59）
- `rss_poll_interval`: RSS轮询间隔（秒）
- `enable_ai_summary`: 是否启用AI总结
- `subscribed_groups`: 订阅群聊列表（文本，每行一个）
- `subscribed_users`: 订阅私聊列表（文本，每行一个）

## 🔄 核心业务流程

### 1. 手动获取流程
```
用户发送 /ainews → 获取RSS最新文章 → 检查缓存 → 
→ 有缓存：直接返回 → 无缓存：AI总结 → 缓存并返回
```

### 2. 定时推送流程
```
到达设定时间 → 检查今日是否已推送 → 
→ 已推送：等待明天 → 未推送：获取RSS文章 → 
→ 检查文章日期 → 非当日：轮询等待 → 当日：检查是否已推送 → 
→ 已推送：跳过 → 未推送：AI总结 → 推送到所有目标 → 标记已推送
```

### 3. 订阅管理流程
```
群内发送 /ainews_sub → 检查是否已订阅 → 
→ 已订阅：提示 → 未订阅：添加到集合 → 保存到文件 → 确认订阅
```

## 🛡️ 容错与优化

### 1. 错误处理
- RSS获取失败：记录日志，返回None
- AI总结失败：自动回退到原文摘要
- 推送失败：记录日志，不标记已推送（后续重试）
- 文件操作失败：记录日志，使用默认值

### 2. 性能优化
- **缓存机制**: AI总结结果缓存10天，避免重复调用LLM
- **数据裁剪**: 推送记录保留最近30天日期和100篇文章链接
- **异步处理**: 所有网络请求和文件操作使用异步
- **原子写入**: 文件更新使用临时文件和原子替换

### 3. 去重机制
- **日期去重**: 基于文章日期的推送记录
- **链接去重**: 基于文章链接的推送记录
- **双重检查**: 推送前同时检查日期和链接

## 🔧 扩展性设计

- 各功能模块相互独立，便于维护和扩展
- 配置参数集中管理，易于调整
